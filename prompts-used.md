# AI Prompts Used

This document contains representative prompts used during the requirement analysis, system design, implementation, testing, and documentation phases of the Hero Cycles Pricing Engine project.

---

## 1. Problem Analysis

### Prompt 1

Analyze the Hero Cycles pricing engine problem statement and identify the core business requirements, stakeholders, and expected system functionality.

### Prompt 2

What challenges arise when bicycle component prices change over time and configurations depend on those components?

### Prompt 3

List important assumptions that can be made while designing a pricing engine for bicycle configurations.

### Prompt 4

Generate functional requirements and non-functional requirements for a bicycle pricing engine application.

### Prompt 6

## AI Usage Summary

AI-assisted tools were used throughout the project lifecycle for brainstorming, requirement analysis, architecture planning, database design, API design, implementation assistance, UI/UX ideation, testing strategy creation, and documentation support.

All generated suggestions, designs, and code snippets were reviewed, modified, integrated, and validated manually before being included in the final Hero Cycles Pricing Engine solution.
